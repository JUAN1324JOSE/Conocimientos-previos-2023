  • GIT INIT

    El comando git init crea un nuevo repositorio de Git. Puede utilizarse para convertir un proyecto existente y

    sin versión en un repositorio de Git, o para inicializar un nuevo repositorio vacío.

  • GIT ADD

    Mueve los cambios del directorio de trabajo al área del entorno de ensayo. Así puedes preparar una

    ánea antes de confirmar en el historial oficial.

  • GIT LOG

    Muestra las instantáneas confirmadas. Te permite ver el historial del proyecto, filtrarlo y buscar cambios concretos.

  • GIT RESET

    Te yudar a mantener tus confirmaciones con un enfoque muy claro al permitirte deshacer la preparación de los

    cambios que no están relacionados con la siguiente confirmación.

  • GIT CLONE

    muestra el estado del directorio de trabajo y del área del entorno de ensayo. Permite ver los cambios que se

    preparado, los que no y los archivos en los que Git no va a realizar el seguimiento. El resultado del estado no

    ninguna información relativa al historial del proyecto.
